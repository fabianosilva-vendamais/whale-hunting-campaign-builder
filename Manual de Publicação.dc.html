VendaMais · Whale Hunting Campaign Builder

Manual de Publicação

GitHub · Vercel · Supabase · OpenAI — passo a passo para colocar o sistema na web e, depois, conectar IA e banco de dados.

0. O que você tem hoje

O arquivo index.html (já renomeado, junto com support.js e a pasta assets/) é a interface completa e navegável do sistema — 15 telas, em branco, prontas para teste e demonstração. As ações de IA e banco são simuladas: os botões "Gerar com IA" mostram um aviso.

Há dois caminhos, e eles se complementam: Caminho A (15 minutos) publica a interface na web para validar o fluxo com a equipe. Caminho B (projeto de desenvolvimento) conecta OpenAI, Supabase e uploads reais — é onde entram as chaves.

Caminho A — Publicar a interface (GitHub + Vercel)

Passo 1 · Baixar o projeto

Baixe o ZIP do projeto (card de download no chat) e descompacte. Os arquivos necessários já estão com os nomes certos:

index.html
support.js
assets/logo-vendamais.png

Não mova o support.js nem a pasta assets/ — o sistema depende deles nesses caminhos. Os demais arquivos do projeto (backup do exemplo, este manual) não precisam subir.

Passo 2 · GitHub

  1. Crie uma conta em github.com (se ainda não tiver).
  2. Clique em New repository → nome whale-hunting-campaign-builder → marque Private → Create repository.
  3. Na página do repositório, clique em uploading an existing file, arraste index.html, support.js e a pasta assets, e clique em Commit changes.

Passo 3 · Vercel

  1. Crie conta em vercel.com com Continue with GitHub (já conecta os dois).
  2. Clique em Add New… → Project → selecione o repositório → Import.
  3. Não mude nada (Framework Preset: Other) → Deploy.
  4. Em ~1 minuto a Vercel entrega a URL pública, ex.: whale-hunting-campaign-builder.vercel.app.

Para atualizar depois: substitua os arquivos no GitHub (novo commit) — a Vercel republica sozinha. Para restringir acesso, use Settings → Deployment Protection na Vercel (recurso pago); o repositório privado protege o código, não a URL.

Caminho B — Conectar IA e banco (versão funcional)

Arquitetura recomendada: Next.js na Vercel + Supabase + OpenAI. Exige um desenvolvedor (ou o Claude Code), usando o protótipo como especificação de telas e fluxo.

Passo 1 · Supabase (banco + arquivos)

  1. Crie conta em supabase.com → New project → nome whale-hunting → região South America (São Paulo) → guarde a senha do banco.
  2. No SQL Editor, crie as tabelas iniciais (esquema abaixo).
  3. Em Storage, crie um bucket materiais (privado) para PDFs e infográficos.
  4. Copie as credenciais em Settings → API: Project URL, anon key e service_role key.
create table campanhas ( id uuid primary key default gen_random_uuid(), nome text not null, segmento text, objetivo text, publico_alvo text, cargos text, solucao_principal text, solucoes_secundarias text, cta_principal text, restricoes text, status text default 'rascunho', periodo_inicio date, periodo_fim date, criada_em timestamptz default now()); create table materiais ( id uuid primary key default gen_random_uuid(), campanha_id uuid references campanhas(id), tipo text, -- special_report | infografico | radar | contas_alvo arquivo_url text, texto_extraido text, observacoes text); create table pecas ( id uuid primary key default gen_random_uuid(), campanha_id uuid references campanhas(id), tipo text, -- analise | lp | regua | post | anuncio | roteiro conteudo jsonb, versao int default 1, status text default 'rascunho'); create table aprovacoes ( id uuid primary key default gen_random_uuid(), peca_id uuid references pecas(id), aprovador text, acao text, comentario text, criada_em timestamptz default now()); create table resultados ( id uuid primary key default gen_random_uuid(), campanha_id uuid references campanhas(id), semana date, metricas jsonb);

Passo 2 · OpenAI — onde colocar a chave

  1. Acesse platform.openai.com → menu API keys → Create new secret key → copie a chave (sk-…). Ela só aparece uma vez.
  2. Em Billing, adicione cartão e um limite mensal (ex.: US$ 20) — sem billing a API não responde.
  3. Na Vercel, dentro do projeto → Settings → Environment Variables → cadastre as variáveis abaixo.
Variável Valor Onde é usada
OPENAI_API_KEYsk-…Só no servidor (rotas de API)
NEXT_PUBLIC_SUPABASE_URLProject URLNavegador + servidor
NEXT_PUBLIC_SUPABASE_ANON_KEYanon keyNavegador + servidor
SUPABASE_SERVICE_ROLE_KEYservice_role keySó no servidor
Regra de ouro: a chave da OpenAI nunca vai no HTML, no JavaScript do navegador ou no GitHub. Ela vive apenas como variável de ambiente na Vercel, e as chamadas acontecem numa rota de API no servidor (ex.: /api/gerar-analise). O navegador chama a sua rota; a sua rota chama a OpenAI.

Passo 3 · O papel de cada serviço

GitHubGuarda o código e o histórico de versões
VercelHospeda o site e executa as rotas de API (onde a OpenAI é chamada)
SupabaseBanco (campanhas, peças, aprovações, resultados) + storage dos PDFs
OpenAIGera análise estratégica, blocos da LP, régua, posts, copies e avaliação da peça da agência

Passo 4 · Ordem recomendada

  1. Publicar o protótipo (Caminho A) e validar o fluxo com a equipe.
  2. Criar o Supabase e as tabelas.
  3. Desenvolver a versão Next.js começando por Nova campanha → Materiais → Análise estratégica; depois LP Builder e exportação; depois o resto.
  4. Conectar a OpenAI uma rota por vez, sempre no padrão: IA gera → humano edita → humano aprova.

Segurança — checklist rápido

  • Repositório GitHub privado
  • Nenhuma chave (sk-…, service_role) commitada no código — só Environment Variables da Vercel
  • Arquivo .env.local no .gitignore (no projeto Next.js)
  • Bucket de materiais do Supabase privado — os PDFs são material estratégico
  • Formulários das LPs com consentimento LGPD (já previsto no bloco Formulário)
  • Limite de gasto mensal configurado na OpenAI

Problemas comuns: página em branco na Vercel → confira se o arquivo se chama index.html e se support.js e assets/ subiram juntos. Erro 429 da OpenAI → limite de billing. Para salvar este manual em PDF: Ctrl+P / Cmd+P → Salvar como PDF.
