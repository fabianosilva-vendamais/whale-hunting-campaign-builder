VendaMais · Whale Hunting Campaign Builder

Colocar o Sistema no Ar

Guia passo a passo do sistema funcional — o que está na pasta sistema/. Sem programação: só subir arquivos e preencher 4 variáveis.

Como este sistema funciona (importante)

Ele tem duas partes que se conversam sozinhas: a tela (o que você vê) e duas rotas de servidor — uma fala com a OpenAI, outra com o Supabase. As suas chaves ficam só no servidor da Vercel; nunca aparecem no navegador. Por isso é seguro.

Você não precisa entender o código. Precisa: (1) subir os arquivos, (2) colar 4 variáveis na Vercel, (3) rodar o SQL no Supabase. Só isso.

Passo 1 · Subir a pasta no GitHub

  1. Baixe o projeto (card de download no chat) e localize a pasta sistema/. Só o conteúdo dela vai para o ar.
  2. Em github.com, crie um repositório novo: whale-hunting-sistema → Private → Create.
  3. Clique em uploading an existing file e arraste o conteúdo da pasta sistema/ — ou seja, o arquivo index.html, o app.jsx e a pasta api (com generate.js e db.js dentro).
  4. Commit changes.
index.html
app.jsx
api/generate.js

Esses 4 arquivos ficam na raiz do repositório (não dentro de outra pasta). A estrutura acima é exatamente o que a Vercel espera.

Passo 2 · Rodar o SQL no Supabase

No seu projeto Supabase → SQL Editor → New query → cole o bloco abaixo → Run. Cria as tabelas uma única vez.

create table if not exists campanhas ( id uuid primary key default gen_random_uuid(), nome text, segmento text, objetivo text, publico_alvo text, cargos text, solucao_principal text, solucoes_secundarias text, cta_principal text, restricoes text, status text default 'rascunho', periodo_inicio date, periodo_fim date, criada_em timestamptz default now()); create table if not exists materiais ( id uuid primary key default gen_random_uuid(), campanha_id uuid references campanhas(id), tipo text, texto_extraido text, observacoes text); create table if not exists pecas ( id uuid primary key default gen_random_uuid(), campanha_id uuid references campanhas(id), tipo text, conteudo jsonb, versao int default 1, status text default 'rascunho');

Não precisa criar Storage nesta versão — os materiais são colados como texto (mais confiável que ler PDF). Storage pode entrar depois.

Passo 3 · Publicar na Vercel

  1. Em vercel.com → Add New… → Project → importe o repositório whale-hunting-sistema.
  2. Framework Preset: Other. Não mude mais nada.
  3. Antes de clicar em Deploy, abra Environment Variables e cadastre as 3 abaixo.
  4. Clique em Deploy. Em ~1 minuto sai a URL.
Variável (nome exato) O que colar
OPENAI_API_KEYSua chave da OpenAI (sk-…)
SUPABASE_URLSupabase → Settings → API → Project URL
SUPABASE_SERVICE_ROLE_KEYSupabase → Settings → API → chave service_role (a secreta)
Atenção aos nomes: use exatamente SUPABASE_URL e SUPABASE_SERVICE_ROLE_KEY — sem o prefixo NEXT_PUBLIC. Se você já tinha criado variáveis com NEXT_PUBLIC, crie estas novas com os nomes certos (pode manter as antigas, não atrapalham).

Se você adicionar as variáveis depois do primeiro deploy, vá em Deployments → ⋯ → Redeploy para elas passarem a valer.

Sobre acesso: o sistema abre direto, sem login — como você pediu. A URL não é divulgada, então fica restrita a quem tem o link. Se um dia quiser uma camada de privacidade sem login interno, ative Vercel → Settings → Deployment Protection → Password Protection (recurso pago): a própria Vercel pede uma senha antes de abrir o site.

Passo 4 · Usar

  1. Abra a URL da Vercel — o sistema já entra no painel de campanhas.
  2. Nova campanha: preencha tema, objetivo, cargos, CTA e salve.
  3. Materiais: cole o texto do Special Report (e Radar/contas, se tiver) e salve.
  4. Estratégia, Landing page, Régua, Conteúdo: clique em Gerar — a IA cria o conteúdo real; edite e salve.
  5. Landing page → Baixar HTML gera o arquivo para colar no WordPress. Exportar baixa a campanha inteira em JSON.

Se algo der errado

Erro ao gerar / OpenAIVerifique OPENAI_API_KEY e o billing/limite na OpenAI (erro 429 = sem crédito).
Erro no bancoRode o SQL do Passo 2 e confira SUPABASE_URL e SUPABASE_SERVICE_ROLE_KEY.
Página em brancoConfira se os 4 arquivos estão na raiz do repositório (index.html, app.jsx, api/generate.js, api/db.js).
Limites honestos desta versão

· Os materiais entram como texto colado (a leitura automática de PDF fica para depois).

· A IA é a OpenAI (sua chave). O modelo padrão é econômico (gpt-4o-mini); dá para trocar criando a variável OPENAI_MODEL.

· A segurança é por senha de equipe — suficiente para dois usuários. Se um dia o time crescer, o próximo passo é ativar o login por e-mail do Supabase (Auth).

Para salvar este guia em PDF: Ctrl+P / Cmd+P → Salvar como PDF.
